Project overview

Build a single-page marketing landing page for Surface, a B2B SaaS product that gives managers honest, anonymized 360-degree team feedback and turns it into actionable insights and development plans. The target audience is team leads and managers at mid-size companies. The tone is professional, direct, and modern — think Linear, Vercel, or Stripe in restraint and quality. No AI-generated imagery, no glowing blobs, no gradient buttons, no purple neon effects.

Design direction

Color palette: The same as current.

Typography: use a clean, modern sans-serif. Suggested pairing — General Sans or Satoshi (via Fontshare) for body and subheadings, and a slightly heavier weight of the same font for headings. Do not use display-only fonts or anything decorative.

Surfaces: cards should use subtle dark surface elevation (slight background shift, no colored borders). No thick colored side borders on any card.

Borders: alpha-blended, not solid gray — oklch(... / 0.10) style.

Shadows: dark-mode-appropriate, low-opacity, warm-toned.

Radius: consistent throughout. Cards use --radius-lg, inputs and buttons use --radius-full for pill shape, matching the existing mockup style.

No AI stock photography anywhere. Where image assets are needed, leave clearly labeled placeholder containers with a comment like <!-- REPLACE: upload /assets/moreno.jpg here --> so the developer can drop in real photos later.

No decorative background shapes, floating blobs, circuit board patterns, or geometric network lines.

Technical notes

Single HTML file with embedded CSS and JS, no build tools required.

All image assets referenced as relative paths from an /assets/ folder. Every image tag should have a descriptive alt, width, height, and loading="lazy".

Where images are not yet available, use a <div class="img-placeholder"> with the asset path as a data-src attribute and a visible label so the developer knows exactly what to drop in.

Light and dark mode toggle in the navbar (sun/moon icon). Default to dark.

Smooth scroll behavior. Navbar shrinks slightly on scroll.

All interactive elements must have visible focus states and meet WCAG AA contrast.

No localStorage or sessionStorage.

JS should be minimal — only used for the scroll navbar effect, theme toggle, and any subtle entrance animations.

Entrance animations: elements fade in and translate up slightly on scroll into view. Keep it subtle — 300ms, opacity and translateY only. Respect prefers-reduced-motion.
